---
title: "Update User Profile"
api: "PUT /users"
description: "Update the authenticated user's profile information"
---

## Authentication

This endpoint requires a valid JWT Bearer token. The update applies to the authenticated user's profile.

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
  
  Format: `Bearer <your_access_token>`
</ParamField>

## Request Body

All fields are optional. Include only the fields you want to update.

<ParamField body="email" type="string">
  User's email address
  
  Must be a valid email format and unique across all users.
  
  Example: `user@example.com`
</ParamField>

<ParamField body="name" type="string">
  User's full name
  
  Example: `John Doe`
</ParamField>

<ParamField body="username" type="string">
  User's unique username
  
  Example: `johndoe`
</ParamField>

<ParamField body="image_path" type="string">
  Path to user's profile image
  
  Example: `/path/to/image.jpg`
</ParamField>

## Response

<ResponseField name="id" type="string">
  Unique user identifier
</ResponseField>

<ResponseField name="email" type="string">
  Updated email address
</ResponseField>

<ResponseField name="name" type="string">
  Updated full name
</ResponseField>

<ResponseField name="username" type="string">
  Updated username
</ResponseField>

<ResponseField name="password" type="string">
  Hashed password (unchanged)
</ResponseField>

<ResponseField name="salt" type="string">
  Password salt (unchanged)
</ResponseField>

<ResponseField name="image_path" type="string">
  Updated profile image path
</ResponseField>

<ResponseField name="role_id" type="number">
  User's role identifier (unchanged)
</ResponseField>

<ResponseField name="created_at" type="string">
  Account creation timestamp (unchanged)
</ResponseField>

<ResponseField name="updated_at" type="string">
  Last update timestamp (reflects current update)
</ResponseField>

## Error Responses

| Status Code | Description |
|-------------|-------------|
| 400 | Bad Request - No fields provided to update |
| 401 | Unauthorized - Invalid or missing JWT token |
| 409 | Conflict - Email already exists for another user |

## Notes

- At least one field must be provided in the request body
- Email addresses must be unique across all users
- The password cannot be updated via this endpoint (use the change password endpoint instead)
- Only the authenticated user can update their own profile
