---
title: 'Get User Details'
api: 'GET /admin/user/{id}'
---

Retrieves detailed information about a specific user by their ID. This endpoint also allows updating user information.

## Get User by ID

### Authentication

Requires admin authentication via Bearer token.

### Path Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the user
</ParamField>

<CodeGroup>
```bash curl
curl -X GET https://api.falconalert.com/admin/user/1 \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN"
```
</CodeGroup>

### Response

<ResponseField name="id" type="number">
  Unique identifier for the user
</ResponseField>

<ResponseField name="name" type="string">
  Full name of the user
</ResponseField>

<ResponseField name="email" type="string">
  User's email address
</ResponseField>

<ResponseField name="username" type="string">
  User's unique username
</ResponseField>

<ResponseField name="password" type="string">
  Hashed password
</ResponseField>

<ResponseField name="salt" type="string">
  Password salt for hashing
</ResponseField>

<ResponseField name="created_at" type="string">
  ISO 8601 timestamp of user creation
</ResponseField>

<ResponseField name="updated_at" type="string">
  ISO 8601 timestamp of last update
</ResponseField>

<ResponseField name="image_path" type="string">
  Path to user's profile picture
</ResponseField>

<ResponseField name="role_id" type="number">
  User's role identifier (1 = admin, 2 = user)
</ResponseField>

<ResponseExample>
```json 200 - Success
{
  "id": 1,
  "name": "Arturo Utrilla",
  "email": "arturo@gmail.com",
  "username": "Arturo_utrilla",
  "password": "e28dce46d564d38b9b58f6c85ff4bb75ba3df1202991f5b6e1d5e5d585c1b8e7",
  "salt": "9tuubdqpqei",
  "created_at": "2025-09-25T12:14:44.000Z",
  "updated_at": "2025-10-24T01:45:55.000Z",
  "image_path": "profile-pictures/d84c32b6211caf458c18f49db1992bebbf5dbfccc581e47aad0b77eee3ea637a.jpg",
  "role_id": 1
}
```

```json 404 - Not Found
{
  "message": "User with id 442 not found",
  "error": "Not Found",
  "statusCode": 404
}
```
</ResponseExample>

## Update User

<Api method="PUT" endpoint="/admin/user/{id}" />

Modifies specific fields of a user by their ID.

### Path Parameters

<ParamField path="id" type="string" required>
  The unique identifier of the user to update
</ParamField>

### Body Parameters

<ParamField body="email" type="string">
  New email address for the user
</ParamField>

<ParamField body="name" type="string">
  New name for the user
</ParamField>

<ParamField body="username" type="string">
  New username for the user
</ParamField>

<ParamField body="image_path" type="string">
  New profile picture path
</ParamField>

<CodeGroup>
```bash curl
curl -X PUT https://api.falconalert.com/admin/user/1 \
  -H "Authorization: Bearer YOUR_ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "John Doe",
    "email": "john.doe@example.com"
  }'
```
</CodeGroup>

### Response

<ResponseField name="id" type="string">
  User's unique identifier
</ResponseField>

<ResponseField name="email" type="string">
  Updated email address
</ResponseField>

<ResponseField name="name" type="string">
  Updated name
</ResponseField>

<ResponseField name="username" type="string">
  User's username
</ResponseField>

<ResponseField name="image_path" type="string">
  Path to profile picture
</ResponseField>

<ResponseField name="role_id" type="number">
  User's role identifier
</ResponseField>

<ResponseExample>
```json 200 - Success
{
  "id": "1",
  "email": "john.doe@example.com",
  "name": "John Doe",
  "username": "johndoe",
  "image_path": "/path/to/image.jpg",
  "role_id": 1
}
```

```json 400 - No Fields Provided
{
  "message": "No fields provided to update",
  "error": "Bad Request",
  "statusCode": 400
}
```

```json 400 - Invalid Email
{
  "message": [
    "email must be an email"
  ],
  "error": "Bad Request",
  "statusCode": 400
}
```

```json 404 - User Not Found
{
  "message": "User with id 442 not found",
  "error": "Not Found",
  "statusCode": 404
}
```

```json 409 - Email Conflict
{
  "message": "Email already exists",
  "error": "Conflict",
  "statusCode": 409
}
```
</ResponseExample>
